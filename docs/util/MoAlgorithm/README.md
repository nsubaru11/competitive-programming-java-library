# MoAlgorithm

## 概要

区間クエリをオフラインで平方分割により高速処理するMo's Algorithmを提供します。

## 実装クラス

### [MoAlgorithm](../../../src/lib/util/MoAlgorithm.java)

- **用途**：
	- オフライン区間クエリ（区間 [l, r) に関する集計値）の一括処理
- **特徴**：
	- クエリは `int[][] lr`（各要素が閉区間 `{l, r}`）で受け取り、`l1 r1 / l2 r2 / ... / lq rq` 形式の入力を一括読み込みした結果をそのまま渡せる
	- クエリを `l` のブロック番号でソートし、ブロックの偶奇で `r` の昇順/降順を切り替える「ジグザグ」順序によりポインタ移動量を削減
	- ソートキー（ブロック番号・ジグザグ用の `r`・クエリ番号）は1つの `long` に詰めてプリミティブソートすることで、比較関数によるオートボクシングのオーバーヘッドを回避
	- 区間への要素の追加・削除処理 (`add`/`remove`) と、現在の区間に対する回答の取得処理 (`answer`) を関数として渡す汎用設計
	- 回答が `long` の場合と任意の参照型 `T` の場合のオーバーロードを用意
- **時間計算量**：
	- O((N + Q) sqrt(N))（`add`/`remove`/`answer` がそれぞれ O(1) の場合）
- **空間計算量**：
	- O(N + Q)

## アルゴリズム（データ構造）選択ガイド

- 区間 [l, r) に対するクエリが事前に全て与えられており（オフライン）、かつ「要素の追加・削除」によって状態を差分更新できる場合に有効です。
- 区間和や区間最小値のように分配則・結合則が成り立つ単純な集計には SegmentTree のほうが適しています。Mo's Algorithm は「区間内の値の種類数」「区間内で同じ値を持つペアの個数」など、差分更新でしか扱いにくい集計に向いています。
- クエリのオンライン処理（クエリごとに即座に回答が必要）が必須の場合は使用できません。

## 注意事項

- クエリは閉区間 [l, r]（0-indexed、両端を含む）として渡してください。例えば1-indexedの入力 [L, R] が与えられた場合は `l = L - 1`, `r = R - 1` に変換してください。
- クエリは少なくとも1つ与えられている前提です（`lr.length >= 1`）。
- N, Q はいずれも 2^20（約 1.0 × 10^6）未満を想定しています。通常の競技プログラミングの制約（N ≦ 10^6 程度）であれば問題ありませんが、それを超える場合はソートキーに用いるビット幅（`IDX_BITS`）を拡張する必要があります。
- `add`/`remove`/`answer` 内で参照する集計用の状態（頻度配列・現在の答えなど）は、ラムダ式から書き換えられるよう配列やクラスフィールドに持たせてください（プリミティブの再代入はラムダから直接行えないため）。
- `add` と `remove` は左右どちらの端からの追加・削除でも同じ処理になることを前提としています。左右で処理を変える必要がある場合（非可換な集計など）は本実装をベースに `addLeft`/`addRight`/`removeLeft`/`removeRight` に分割してください。
