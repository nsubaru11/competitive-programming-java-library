# IntArray2D / LongArray2D 利用ガイド

## 概要

[`IntArray2D`](../../../src/lib/ds/arrays/IntArray2D.java) と [`LongArray2D`](../../../src/lib/ds/arrays/LongArray2D.java) は、`h * w` 要素を一次元配列へ圧縮して保持する2次元配列です。90度単位の論理回転ではデータを移動せず、座標変換によって O(1) でアクセスします。

## 特徴

- 多次元配列ではなく連続した一次元プリミティブ配列で保持
- 左右90度回転をカウンタ更新だけで実行
- 回転状態を反映した `get` / `set`
- 正方行列の論理転置トグルに対応
- 物理配列順を低コストで走査するプリミティブ iterator
- `int` / `long` で同じ座標 API を提供

## 依存関係

- `java.util.PrimitiveIterator`
- `java.util.function.IntBinaryOperator`
- `java.util.function.LongBinaryOperator`

## 主な機能（メソッド一覧）

### 1. コンストラクタ

| メソッド                                                 | 戻り値の型 | 説明                           |
|------------------------------------------------------|-------|------------------------------|
| `IntArray2D(int h, int w, IntBinaryOperator init)`   | -     | `init(i, j)` で int の各要素を初期化  |
| `LongArray2D(int h, int w, LongBinaryOperator init)` | -     | `init(i, j)` で long の各要素を初期化 |

### 2. アクセス・更新

| メソッド                       | 戻り値の型          | 説明                   |
|----------------------------|----------------|----------------------|
| `get(int i, int j)`        | `int` / `long` | 現在の回転・転置状態で論理座標の値を返す |
| `set(int i, int j, value)` | `void`         | 現在の回転・転置状態で論理座標を更新   |

### 3. 論理変換

| メソッド          | 戻り値の型  | 説明                     |
|---------------|--------|------------------------|
| `lRotate()`   | `void` | 反時計回りに90度論理回転          |
| `rRotate()`   | `void` | 時計回りに90度論理回転           |
| `transpose()` | `void` | `h == w` の場合だけ転置状態をトグル |

### 4. 反復

| メソッド         | 戻り値の型                                | 説明                |
|--------------|--------------------------------------|-------------------|
| `iterator()` | `PrimitiveIterator.OfInt` / `OfLong` | 初期化時の物理配列順で全要素を走査 |

## 利用例

```java
IntArray2D grid = new IntArray2D(2, 3, (i, j) -> i * 10 + j);
// 0  1  2
// 10 11 12

grid.lRotate();
// 論理表示:
// 2 12
// 1 11
// 0 10

System.out.println(grid.get(0, 1)); // 12
grid.set(1, 1, 99);
```

## 注意事項

- `h`, `w`, `h * w` が配列として確保可能な問題制約で使用します。
- `get` / `set` は現在の論理サイズ内の座標で呼び出します。
- 90度または270度回転中の論理サイズは `w x h`、0度または180度では `h x w` です。
- 高さ・幅の getter はないため、利用側で元の `h`, `w` と回転回数を管理します。
- `transpose()` は正方行列だけに作用し、長方形では何もしません。
- 回転カウンタと転置フラグは独立して保持されます。両操作を複合する場合は必要な座標対応を確認して使用します。
- `iterator()` は論理回転・転置を反映せず、物理配列順で返します。

## パフォーマンス特性

- 構築: O(h w)
- `get`, `set`, `lRotate`, `rRotate`, `transpose`: O(1)
- iterator による全走査: O(h w)
- 使用メモリ: O(h w)

### バージョン管理について

バージョン番号は2桁で管理します：

- 1桁目（メジャーバージョン）: メソッドの追加や機能拡張があった場合に更新
- 2桁目（マイナーバージョン）: 誤字修正、バグ修正、マイクロ高速化などの小さな更新があった場合に更新
